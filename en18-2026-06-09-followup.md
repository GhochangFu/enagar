# EN-18 follow-up (2026-06-09): "no previous applications" + "application_id is not valid"

## Symptom (citizen 9836177767)

- Citizen PWA showed `Applications 0`, `Payments 0`, `Grievances 23`.
- Creating a new community-hall booking failed with `application_id is not valid for this municipality`.

## Root cause

Two stacked bugs, both triggered by running the compiled API from the **repo root**
(`node apps/api/dist/main`):

1. `apps/api/src/load-infra-env.ts` resolved `infrastructure/.env` as
   `cwd/../../infrastructure/.env`. With `cwd = D:\UDProjects\MunicipalServices`
   that path is `D:\infrastructure\.env`, which does not exist. The previous
   `pnpm dev` flow used `cwd = apps/api`, so the loader worked there.
   - Result: `process.env.DATABASE_URL` was never set in the running process.
2. `apps/api/src/modules/applications/applications.module.ts` picked
   `InMemoryApplicationStore` whenever `APPLICATION_STORE_PROVIDER` was not
   set. With the env loader failing, the API held an empty in-memory list
   even though rows existed in Postgres.
   - Result: `applicationsService.findAll/findOne` returned `[]`/404 to the
     PWA, and `bookings.service.ts:linkApplicationToHold` could not resolve
     the application id, producing the "application_id is not valid for this
     municipality" error.

## Fix

1. `apps/api/src/load-infra-env.ts` — also try `cwd/infrastructure/.env`
   (repo-root layout) in addition to `cwd/../../infrastructure/.env`
   (`apps/api` layout). First match wins.
2. `apps/api/src/modules/applications/applications.module.ts` — change the
   `APPLICATION_STORE` factory default: pick `PostgresApplicationStore` when
   `DATABASE_URL` is set (or when the caller explicitly requested `postgres`).
   `InMemoryApplicationStore` is now only used when
   `APPLICATION_STORE_PROVIDER=in-memory` _or_ no `DATABASE_URL` exists.
3. `apps/api/src/modules/applications/phase2-api.integration.spec.ts` and
   `apps/api/src/modules/payments/payment-portal.http.integration.spec.ts` —
   explicitly set `process.env.APPLICATION_STORE_PROVIDER = 'in-memory';` in
   `beforeAll` so they keep their original in-memory isolation.
