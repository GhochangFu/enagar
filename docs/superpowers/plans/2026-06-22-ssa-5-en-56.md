## Exit Criteria

SSA-5 is complete when all are true:

1. **Full scope:** Steps 1→5 navigable with Back/Next; Next skips already-complete layers in full scope.
2. **Skip to review:** Available for partial scopes when current step is complete; for full scope when steps 1–4 are complete.
3. **Open designer:** Deep link from every active step.
4. **Review:** Shows draft-layer readiness separate from publish readiness (`ready_to_publish`).
5. **Regenerate:** Workflow step UI states that workflow edits do not clear form/payment drafts.
6. **Session sync:** After chat turn, session `step_completion` and `archetype` refresh in UI.
7. **Verification:** Flow unit tests pass; existing SSA specs remain green.
8. **Review:** No open high-severity spec compliance issues.

## Manual UI Tests (operator)

**Prerequisites:** API + admin-tenant running, LLM configured, service with partial drafts acceptable.

### Full scope pilot

1. Start **Full setup** → confirm header shows `Step 1 of 5 — Intent & archetype`.
2. Complete intent (archetype + requirements) → step 1 shows ✓.
3. Click **Next** → lands on step 2 (or first incomplete step if 2 already ✓).
4. Walk through form → workflow → payment using chat; use **Back** to return without losing drafts.
5. On workflow step, confirm note: form/payment drafts are preserved when regenerating workflow.
6. Reach step 5 → **Draft layers ready** and **Ready to publish** show distinct values.
7. Use **Open full designer** → Service Designer opens; publish manually.

### Partial scope skip

8. **Form only** → complete form step → **Skip to review** enabled → lands on step 5.
9. **Workflow only** → after template apply, skip to review works.

### Regenerate safety

10. In full scope, set form + payment, then on step 3 replace workflow → form preview and payment summary unchanged.
