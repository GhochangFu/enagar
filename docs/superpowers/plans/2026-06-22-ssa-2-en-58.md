# SSA-2 (EN-58) ‚Äî Service Setup Assistant Form Step Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task.

**Goal:** Deliver the Form step for Service Setup Assistant ‚Äî typed LLM tools for tenant and state admin, SSE chat orchestration, form preview UI, and auto-save drafts via existing designer APIs.

**Architecture:** Extend `service-setup-assistant` with a per-step tool registry, tenant/state form tools calling `AdminTenantService` / `AdminStateService`, and `ServiceSetupAssistantService` streaming SSE via reused `ChatbotLlmService`.

**Tech Stack:** NestJS, Prisma, Next.js 14, `@enagar/forms`, existing chatbot LLM adapter (ADR-0008), Jest.

**Jira:** [EN-58](https://ghochangfu.atlassian.net/browse/EN-58) ¬∑ Parent [EN-51](https://ghochangfu.atlassian.net/browse/EN-51)

**Depends on:** SSA-1 (EN-53) ‚Äî sessions, readiness, wizard shell.

## Exit Criteria

SSA-2 is complete when all are true:

1. **Tool registry:** `getToolsForStep(persona, scope, step)` returns only allowed tools for form step (step 2).
2. **Tenant tools:** `applyFormDraft`, `loadGlobalTemplate`, `proposeFormFields` validate before write; wrong `service_code` rejected.
3. **State tools:** `applyGlobalFormSchema`, `proposeGlobalFormFields` validate and patch global template; `state_admin` guard enforced.
4. **SSE:** `POST .../message` streams `meta`, `token`, `tool_result`, `draft_updated`, `done`, `error` events per `SetupAssistantSseEvent`.
5. **Persistence:** User/assistant messages and tool audit rows written per turn.
6. **DPA:** Reuses `ChatbotLlmService` with `SETUP_ASSISTANT_SKIP_DPA_DEV` for local dev (ADR-0016).
7. **UI tenant:** Form-only scope E2E ‚Äî chat sends message, preview refreshes on `draft_updated`, step 2 marked complete when `form_draft_valid` is green.
8. **UI state:** Form assistant shell at `/dashboard/library/[code]/form/setup-assistant` with chat + preview.
9. **Verification:** New API specs pass; mocked LLM integration test green.
10. **Review:** Spec compliance + code quality review with no open high-severity issues.

## Manual UI Tests (Your side)

**Prerequisites:** `pnpm --filter @enagar/api prisma:migrate:deploy`, API + admin apps running, `SETUP_ASSISTANT_SKIP_DPA_DEV=true` in dev, LLM provider configured (`LLM_PROVIDER` / Ollama local).

### Tenant (`admin-tenant`)

1. Open Setup Assistant ‚Üí **Form only** scope ‚Üí confirm step 2 active.
2. Send chat message: _"Add a text field called applicant name"_ ‚Äî verify streaming reply and tool result in UI.
3. Confirm form preview updates after `draft_updated` without manual refresh.
4. Readiness sidebar: `form_draft_valid` turns green after valid draft applied.
5. Reload page with `?sessionId=` ‚Äî chat history loads (if history endpoint used) and preview shows saved draft.
6. Open Service Designer ‚Äî confirm form draft matches assistant changes.

### State (`admin-state`)

7. Open Global Form Builder ‚Üí **Open Setup Assistant** link.
8. Create form session; send message to add a field; verify global template preview updates.
9. Return to Global Form Builder ‚Äî saved draft reflects assistant changes.

### Security

10. Tampered `sessionId` or cross-tenant service ID ‚Üí request rejected.
11. Non-`state_admin` token on state routes ‚Üí 403.
