# SSA-1 (EN-53) — Service Setup Assistant Foundation Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Deliver the foundation slice of Service Setup Assistant: session persistence, readiness checklist, tenant admin setup-assistant API routes, and tenant wizard shell (no LLM chat orchestration yet).

**Architecture:** Add a new API module `service-setup-assistant` that composes existing `AdminTenantService` read/write primitives and validators (`validateFormSchema`, `validateWorkflowDefinition`), plus a thin `admin-tenant` UI shell that creates and navigates setup sessions by scope.

**Tech Stack:** NestJS, Prisma/PostgreSQL, Next.js App Router, Jest, existing tenant admin auth/session utilities.

**Jira:** [EN-53](https://ghochangfu.atlassian.net/browse/EN-53)

## Exit Criteria

SSA-1 is complete when all are true:

1. **Types:** `SetupAssistantScope`, `SetupSessionDto`, readiness and SSE event types compile and are exported by `@enagar/types`.
2. **Persistence:** `service_setup_sessions`, `service_setup_messages`, and `service_setup_audit_logs` exist in schema and migration.
3. **Session service:** scope-to-step mapping is correct (`full=1`, `form=2`, `workflow=3`, `payment=4`, `review=5`) and cross-tenant session access is rejected.
4. **Readiness service:** returns checklist with `green/amber/red` signals and `ready_to_publish` aggregate computed from current service drafts + published state.
5. **Tenant routes:** create/get session, patch step, and readiness endpoints work with tenant portal auth.
6. **Wizard shell:** scope picker creates a session and displays step progress + readiness panel without LLM chat dependency.
7. **Verification:** API tests and tenant typecheck/tests pass for touched paths.
8. **Review:** spec compliance + code quality review complete with no open high-severity issues.
