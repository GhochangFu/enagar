## Exit Criteria

SSA-4 is complete when all are true:

1. **Registry:** Step 4 + scope `full`/`payment` exposes only config tools; step 5 exposes only review tools; step 1 (`full` only) exposes intent tools.
2. **Config tools:** `applyServiceConfig` validates fee JSON, documents, governance policies; rejects unknown `revenue_head_code`.
3. **Masters context:** Revenue heads listed in Step 4 system prompt before each message.
4. **Intent:** `detectArchetype` and `summarizeRequirements` persist to session only — no form/workflow/config PATCH in step 1.
5. **Review:** `getReadinessChecklist`, `explainBlockers`, `previewCitizenForm` are read-only.
6. **SSE:** Tenant `POST .../message` works on steps 1, 4, 5; step 4 emits `draft_updated` with layer `config`.
7. **Step completion:** Step 4 complete when `config_complete` is green; step 1 when archetype + requirements set; step 5 when draft readiness items are green.
8. **UI:** Payment-only scope E2E — chat on step 4, config summary panel; review step shows checklist with designer links.
9. **Verification:** New specs pass; existing SSA specs remain green.
10. **Review:** No open high-severity spec compliance issues.

## Manual UI Tests (operator)

**Prerequisites:** API + admin-tenant running, `SETUP_ASSISTANT_SKIP_DPA_DEV=true`, LLM configured, service with fee-bearing config potential.

### Payment scope

1. Open Setup Assistant → **Payment only** → confirm step 4 active, chat enabled.
2. Ask: _"Set a fixed fee of ₹50 and require parent Aadhaar PDF"_ → `applyServiceConfig: OK`, config summary updates.
3. Ask: _"Map revenue head cert-fee"_ (use a real code from Masters) → revenue head shown in summary.
4. Readiness: `config_complete` turns green; step 4 shows ✓.
5. Service Designer → config tab matches assistant changes.

### Review scope

6. Open **Review only** → step 5 active.
7. Ask: _"What is blocking publish?"_ → `explainBlockers` lists amber/red items with messages.
8. Checklist panel shows links to Service Designer (`#form`, `#workflow`, config).
9. `ready_to_publish` reflects draft validity (not necessarily published).

### Full scope

10. Start **Full setup** → step 1: describe service purpose → `detectArchetype` + `summarizeRequirements` run.
11. Navigate steps 1→5; payment tools must not run on step 3; workflow tools must not run on step 4.

### Security

12. Cross-tenant session/service → rejected.
